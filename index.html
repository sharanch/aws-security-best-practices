// selected attack scenarios documented

▶ IMDS Credential Theft via SSRF
IMDSv1 allows any process or SSRF-vulnerable app to curl 169.254.169.254 and steal EC2 role credentials — usable from anywhere until TTL expires.
EC2 · IMDSv2 · SSRF
▶ IAM Privilege Escalation via iam:PassRole
A low-privilege role with iam:PassRole + lambda:CreateFunction can deploy a Lambda with an admin execution role and invoke it — full account takeover in two API calls.
IAM · Lambda · Privilege Escalation
▶ S3 Ransomware via Mass Delete
Without versioning and MFA delete, a stolen role with s3:DeleteObject wipes an entire bucket permanently. Recovery is impossible without a separate backup.
S3 · Ransomware · Versioning
▶ Lambda Env Var Credential Harvest
Secrets stored as Lambda environment variables are readable by any code path — including a compromised dependency — and visible in plain text via GetFunctionConfiguration.
Lambda · Secrets Manager · Supply Chain
▶ Attacker Disables CloudTrail First
The first action of a sophisticated attacker after gaining admin access: aws cloudtrail stop-logging. Without an SCP blocking this, all subsequent activity is unlogged.
CloudTrail · SCP · Detection
▶ Publicly Accessible RDS Brute Forced
RDS instances with PubliclyAccessible=true and a security group open on port 5432/3306 are directly reachable from the internet — bypassing the application layer entirely.
RDS · Security Groups · Network
